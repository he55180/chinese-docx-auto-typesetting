## 项目简介

本工具专为解决中文公文 Word 文档格式混乱问题而设计。  
区别于普通格式刷，能够深度识别文档内部格式漏洞，自动修正各类不规范排版问题，  
严格遵循 **GB/T 9704-2012《党政机关公文格式》** 国家标准开发。

**三大特点：**

- **🎯 极简操作** — GUI 界面，拖拽选文件，一键处理，零门槛上手
- **🔒 本地离线** — 所有处理在本机完成，文档内容不联网，涉密文件放心使用
- **📋 标准规范** — 严格对照国标参数，页边距、字体字号、行距缩进全部合规

---

## 核心能力

本工具不只是格式刷，能深度识别并修复以下常见问题：

1. **🔣 标点符号规范化** — 自动检测并修正括号、引号、逗号、句号等全半角混用，统一为中文规范符号（v2.0 新增）
2. **🧹 格式深度清洗** — 清除杂乱字体颜色、不规范下划线/斜体、多余空格，还原干净版面（v2.0 新增）
3. **📏 页边距校准** — 自动统一页边距，符合公文版心要求（天头37mm / 地脚35mm / 左28mm / 右26mm）
4. **🔤 字体智能适配** — 智能识别标题与正文层级，自动匹配黑体、楷体、仿宋等公文专用字体字号
5. **📝 首行缩进补全** — 扫描全文，为缺失首行缩进的段落自动补标准 2 字符缩进
6. **📐 行距统一规范** — 识别全文不统一行距，一键调整为标准固定行距（30pt）
7. **1️⃣ 四级标题体系** — 完整支持 `一、→（一）→1.→(1)` 四级标题自动识别、编号矫正与格式化
8. **🔢 无编号标题补全** — 智能识别短小无标点段落，自动识别为二级标题并补全编号
9. **📊 表格格式化** — 自动格式化表格字体字号，表头黑体、内容仿宋，保留原有结构
10. **🖼️ 图片/表格完整保护** — 内容 100% 保留，位置大小格式不变
11. **📦 页码自动生成** — 在文档页脚居中自动插入页码
12. **📂 批量文件处理** — 多选文件或整个文件夹批量处理，输出目录统一管理（v2.0 新增）

## 使用方法

### GUI 模式（推荐）

#### 第一步：选择文件

点击「**选择文件**」单选/多选 Word 文档，或点击「**选择文件夹**」批量导入整个目录。

#### 第二步：选择处理模式

| 模式 | 适用场景 | 是否修改文件 |
|------|---------|------------|
| **🪄 智能一键处理**（推荐） | 日常通用，标点修正 + 格式清洗 + 公文排版全套 | ✅ 输出新文件 |
| **🩺 格式诊断** | 检查文档有哪些问题，暂不修改 | ❌ 只输出报告 |
| **🩹 仅标点修复** | 只修正标点符号，保留原有排版格式 | ✅ 输出新文件 |

#### 第三步：开始处理

点击「**▶ 开始处理**」按钮。

- 处理完成后，输出文件自动命名为 `原文件名_processed.docx`
- **原始文件永远不会被覆盖或修改，请放心使用**

---

### 命令行模式（进阶）

```bash
# 智能一键处理（默认）
python format_expert.py 你的文档.docx

# 格式诊断（只检测，不修改）
python format_expert.py 你的文档.docx --mode diagnose

# 仅标点修复
python format_expert.py 你的文档.docx --mode punct

# 指定输出路径
python format_expert.py 你的文档.docx -o 排版完成.docx

# 不添加页码
python format_expert.py 你的文档.docx --no-pagenum

# Markdown → Word 管道模式（需安装 Pandoc）
python run_pipeline.py input.md
```

**Markdown 管道流程：**

```
input.md
  │
  ▼  [Step 1] preprocess_md.py（转义数字列表，防 Pandoc 误解析）
  │
  ▼  [Step 2] Pandoc + 黄金模板（Markdown → raw .docx）
  │
  ▼  [Step 3] format_expert.py（GB/T 9704-2012 精排引擎）
  │
  ▼
output_标准排版.docx
```

## 排版标准

严格遵循 **GB/T 9704-2012《党政机关公文格式》**：

| 参数 | 值 |
|------|-----|
| 纸张 | A4（210mm × 297mm） |
| 页面天头 | 37mm |
| 页面地脚 | 35mm |
| 左边距 | 28mm |
| 右边距 | 26mm |
| 行距 | 固定 30pt |
| 大标题 | 方正小标宋简体 22pt 加粗居中 |
| 一级标题 `一、` | 黑体 16pt 加粗 |
| 二级标题 `（一）` | 楷体 16pt |
| 三级标题 `1.` | 仿宋 16pt 加粗 |
| 四级标题 `(1)` | 仿宋 16pt |
| 正文 | 仿宋_GB2312 16pt 首行缩进 2 字符 |
| 表头 | 黑体 10pt 加粗 |
| 表格内容 | 仿宋 10pt |
| 页码 | 仿宋 10pt 居中，页脚位置 |

## 标题识别规则

```
一、标题      → 一级标题（黑体 16pt 加粗）
（一）标题    → 二级标题（楷体 16pt）
1. 标题       → 三级标题（仿宋加粗）
(1) 标题      → 四级标题（仿宋 16pt）
短小无标点段落 → 自动识别为二级标题并补全编号
```

**内容 100% 保护原则：**
- 图片：位置、大小、格式完全不变
- 表格：结构不变，仅规范字体
- 阿拉伯数字编号（`1.`、`21、`、`(1)` 等）：绝不修改原始编号内容

## 字体说明

本工具默认使用以下字体（Windows 系统通常内置）：

| 字体 | 用途 |
|------|------|
| 方正小标宋简体 | 大标题 |
| 黑体 | 一级标题、表头 |
| 楷体 | 二级标题 |
| 仿宋_GB2312 | 正文、三级/四级标题、页码 |

> 如文档打开后字体显示异常，请确保系统已安装上述字体。  
> Linux/macOS 运行时，工具会自动切换为等效中文字体（WenQuanYi、Noto Sans CJK SC）。

## 常见问题

**Q：处理后文档字体显示异常或乱码？**  
A：请确保系统已安装「仿宋_GB2312」「黑体」「楷体」公文专用字体。Windows 通常已内置。

**Q：提示「文件不存在」错误？**  
A：请检查文件路径中是否含有特殊字符。建议将文件放在桌面或纯英文路径下再处理。

**Q：可以批量处理多个文件吗？**  
A：支持。在 GUI 界面点击「选择文件」时多选（Ctrl/Shift），或点击「选择文件夹」批量导入整个目录。

**Q：原文件会被覆盖吗？**  
A：不会。输出文件自动在原文件名后添加 `_processed` 后缀，与原文件并列保存，原始文件始终不变。

**Q：Markdown 管道模式需要什么额外工具？**  
A：需要安装 [Pandoc](https://pandoc.org/)。直接排版 `.docx` 文件无需 Pandoc。

**Q：图片和表格会丢失吗？**  
A：不会。工具对图片和表格有完整保护机制，仅格式化字体，内容和结构 100% 保留。
